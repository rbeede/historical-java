Project # 4

Due Date: March 3, 2000

A car dealership has suffered a computer crash that damaged its inventory database. Entries for each vehicle still exist, including certain specifications, although the actual type of vehicle has been lost. Your job is to develop an application that will take as inputs the price, number of doors, and transmission type, and then output the vehicle series type (A-G). In addition, if the vehicle is a G-series, you must then and only then also ask the user for the horsepower of the vehicle and determine the trim level (1-4). The following table has been constructed detailing the vehicle series and trim levels in inventory and their specifications.
Vehicle Series Trim level Price Doors Transmission Horsepower
A - $12000-$14500 4 Manual -
B - $17500-$19000 4 Automatic -
C - $21500-$24000 4 Manual -
D - $16000-$18500 2 Manual -
E - $11250-$13000 4 Automatic -
F - $18000-$19250 4 Manual -
G 1 $22150-$26200 2 Automatic 200
G 2 $23050-$27100 2 Automatic 215
G 3 $24600-$28750 2 Automatic 240
G 4 $24500-$29200 2 Automatic 310

 
Use at least one switch statement. When you are deciding how to code the if else statements, look for trends so that you can group the data efficiently. Define the following methods in an instantiable class called Inventory.
getSeriesInput() Gets the Price, Doors, and Transmission data.
getHorsepower() Gets the Horsepower data, only called if a G series.
findSeries() Determines the Series.
findTrimLevel() Determines the Trim, only called if a G series.
The main class called InventoryMain will then declare and instantiate an Inventory object.
Something to think about …With valid input, can every vehicle series and (if applicable) trim level be determined?
